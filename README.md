# Shopchik - Telegram WebApp Каталог товаров

Telegram бот с интегрированным WebApp для витрины товаров. Пользователи просматривают каталог, ищут товары, открывают детальную карточку и одним нажатием переходят в чат с менеджером с готовым текстом заявки. Администратор управляет товарами и делает рассылки через веб-интерфейс.

## Функциональные возможности

### Клиентская часть (WebApp)
- Главная страница с поиском товаров по названию
- Список товаров в виде карточек (фото, название, цена)
- Детальная страница товара: несколько фото, полное описание, цена
- Кнопка "Написать по покупке" - открывает чат с менеджером через t.me/username с подготовленным текстом: "Здравствуйте, я хочу купить [название товара]"
- Интеграция с Telegram WebApp SDK (тема, размер экрана)

### Админ-панель
- CRUD операции с товарами: создание, просмотр, редактирование, удаление
- Загрузка нескольких фотографий для каждого товара
- Рассылка сообщений (текст + фото) всем пользователям, которые запускали бота
- Отправка рассылки сейчас или по расписанию
- Доступ только по Telegram ID из переменной ADMIN_IDS

### Бот
- Команда /start открывает WebApp
- Автоматическое сохранение всех пользователей в базу данных для рассылок

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/gregor2252/shopchik.git
cd shopchik
2. Настройка переменных окружения
Скопируйте файл с примером переменных:

bash
cp .env.example .env
Отредактируйте файл .env, обязательно заполните следующие поля:

ini
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_IDS=123456789
DB_PASSWORD=надежный_пароль
Остальные переменные можно оставить по умолчанию.

Как получить токен бота:

Напишите @BotFather в Telegram

Отправьте команду /newbot

Следуйте инструкциям, скопируйте полученный токен

Как получить Telegram ID:

Напишите @userinfobot в Telegram

Нажмите Start

Скопируйте ваш ID

3. Запуск проекта
bash
docker compose up -d --build
4. Проверка работы контейнеров
bash
docker compose ps
Все три сервиса (backend, frontend, db) должны быть в статусе Up.

5. Открытие админ-панели
Откройте в браузере:

text
http://localhost/admin.html?user_id=ВАШ_TELEGRAM_ID
Замените ВАШ_TELEGRAM_ID на ваш Telegram ID, который вы указали в ADMIN_IDS.

В админ-панели вы можете:

Добавлять новые товары (кнопка "Добавить товар")

Редактировать существующие товары

Удалять товары

Создавать рассылки

6. Просмотр каталога
Откройте в браузере:

text
http://localhost
Здесь отображаются все товары для покупателей.

Настройка HTTPS для доступа из Telegram
Telegram Mini App требует HTTPS соединения. Для локальной разработки используйте туннель.

Вариант 1: xTunnel (рекомендуется для России)
Скачайте xTunnel с официального сайта: https://xtunnel.ru

Зарегистрируйтесь и получите лицензионный ключ

Откройте командную строку в папке с xtunnel.exe:

bash
xtunnel.exe register ваш_ключ
xtunnel http 80
Скопируйте полученный HTTPS адрес (вида https://что-то.tunnel4.com)

Вариант 2: ngrok
Скачайте ngrok с официального сайта: https://ngrok.com

Зарегистрируйтесь и получите токен

Выполните авторизацию:

bash
ngrok config add-authtoken ваш_токен
ngrok http 80
Скопируйте полученный HTTPS адрес (вида https://abc123.ngrok.io)

Настройка бота после получения HTTPS адреса
Откройте файл .env и обновите переменные:

ini
WEBAPP_URL=https://полученный_адрес
ADMIN_URL=https://полученный_адрес/admin.html
Перезапустите контейнеры:

bash
docker compose restart
Настройте кнопку WebApp у бота через @BotFather:

Отправьте /mybots

Выберите вашего бота

Нажмите Bot Settings

Нажмите Menu Button

Нажмите Configure Menu Button

В поле URL вставьте полученный HTTPS адрес

Сохраните изменения

Проверьте работу:

Откройте Telegram

Найдите вашего бота

Отправьте /start

Нажмите кнопку "Открыть магазин"
